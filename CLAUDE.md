# О проекте
Это проект по разработке SRE-агента в рамках Инфраструктурного трека дисциплины "LLM & Agentic AI"

# Технологии и инструменты
- Python 3.12+
- Пакетный менеджер: `uv`
- Валидация данных и работа с секретами: `pydantic` (v2)

# Правила написания кода (Code Style & Typing)
- **Строгая типизация:** Все функции, методы и переменные должны иметь аннотации типов. Категорически избегай использования `Any`. Если тип сложный, используй `Union`, `Optional`, `Generic` или `TypeVar`. Если тип от сторонней библиотеки, то используй его.
- **Pydantic:** Используй возможности Pydantic v2 (например, `model_validate`, `model_dump`). Для валидации и описания полей активно применяй `Field`.
- **Документация (Docstrings):** Пиши docstrings для всех публичных модулей, классов и функций. Используй Google Style. Обязательно описывай аргументы (`Args:`) и возвращаемые значения (`Returns:`).
- **Импорты:** Используй абсолютные импорты (например, from app.models.user import User), избегай относительных (from ..models import User), кроме импортов внутри одного модуля. Группируй импорты: сначала стандартная библиотека, потом сторонние (pydantic), потом локальные.

# Правила работы (Workflow)
- **Управление зависимостями:** Используй **только** `uv`. Для добавления пакетов используй `uv add <package>`, для запуска скриптов — `uv run <script>`. Не используй `pip` или `poetry`.
- **Переменные среды и секреты:** Не хардкодь секреты и конфигурацию. Используй pydantic-settings (класс BaseSettings) для управления конфигурацией и читай переменные из файла .env.
- **Актуализация документации:** После любого изменения логики, добавления новой функциональности, изменения API или установки новых зависимостей — **всегда проверяй и обновляй README.md**, чтобы он оставался актуальным.

# Команды проверки (Выполняй перед коммитом)
- Запуск проекта: `uv run python main.py`
