# Высокая загрузка CPU

## Симптомы
- Алерт: CPU usage > 90% for 5 min
- `top` показывает один или несколько процессов с высоким %CPU
- Замедление ответов сервиса

## Диагностика
1. `top -bn1 | head -20` — общая картина и top-процессы
2. `ps aux --sort=-%cpu | head -10` — процессы с наибольшим потреблением CPU
3. `uptime` — load average (1/5/15 мин)
4. `iostat -x 1 3` — проверить, не вызвано ли CPU wait I/O
5. `docker stats --no-stream` — потребление по контейнерам

## Возможные причины
- Зацикленный процесс или утечка в коде
- Высокая нагрузка (пик трафика)
- Фоновый процесс (backup, cron job)
- Недостаточно ресурсов для текущей нагрузки

## Рекомендации
1. Идентифицировать процесс-потребитель через `top` или `ps`
2. Если это приложение — проверить логи на ошибки
3. Если фоновый процесс — проверить crontab и systemd timers
4. При пике трафика — рассмотреть горизонтальное масштабирование
