# Высокое потребление памяти

## Симптомы
- Алерт: Memory usage > 85% for 5 min
- OOM killer в dmesg
- Swap активно используется

## Диагностика
1. `free -h` — общая картина RAM/swap
2. `ps aux --sort=-%mem | head -10` — процессы с наибольшим потреблением RAM
3. `docker stats --no-stream` — потребление по контейнерам
4. `dmesg | tail -50` — проверить OOM killer
5. `cat /proc/meminfo | head -10` — детальная информация

## Возможные причины
- Утечка памяти в приложении
- Кэш, который не очищается
- Слишком много одновременных подключений
- Недостаточно RAM для текущей нагрузки

## Рекомендации
1. Идентифицировать процесс через `ps aux --sort=-%mem`
2. Проверить логи приложения на ошибки
3. Если Redis — проверить `redis-cli INFO memory` и maxmemory policy
4. Рестарт контейнера как временная мера
